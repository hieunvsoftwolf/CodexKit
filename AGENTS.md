For local Codex profile management on this machine:

- The canonical shared custom skill store is `~/.codex/skills`.
- Do not treat `CODEX_HOME/skills` as the canonical install target for reusable custom skills when working across rotating profiles.
- Treat reusable skills and project-owned skills as different classes:
  - reusable shared skills belong in `~/.codex/skills`
  - project-owned generated skills belong inside the target repository
- When asked to install or create reusable custom skills, install them into `~/.codex/skills` first.
- Do not install `control-agent-<project>` or other repo-specific generated skills into `~/.codex/skills`; generate them into the target repo under `.agents/skills/`.
- Profile-specific `skills/` directories should be symlinked to `~/.codex/skills` when the user wants the same skill set across multiple accounts.
